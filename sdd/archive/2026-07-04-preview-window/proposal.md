## 為什麼做

目前 `src\mygo-search.ps1` 只會在終端機輸出 alt 文字與圖片 URL，使用者必須自己複製連結、開瀏覽器才能看到圖片，體驗不佳。參考 `preview-image-sample-code.ps1` 已經示範了用 Windows Forms 彈出視窗預覽單張圖片的做法，可以把這個能力整合進主腳本，讓搜尋/隨機取圖後直接跳出視窗看圖，並提供下載功能。

## 要改什麼

- 執行 `mygo-search.ps1` 搜尋或隨機取圖成功後，彈出一個 Windows Forms 視窗，視窗內以可捲動版面顯示這次取得的所有圖片（可能不只一張）。
- 每張圖片下方顯示對應的 alt 文字，以及一個「下載」按鈕。
- 點擊某張圖片的「下載」按鈕時，跳出標準的「另存新檔」對話框（`SaveFileDialog`），預設檔名可用該圖片的 alt 文字，讓使用者自行選擇儲存路徑與檔名。
- 關閉視窗即代表結束本次指令執行（沿用 `ShowDialog()` 的阻塞行為）。
- 保留原本終端機文字輸出（alt 文字 + URL），視窗是額外附加的功能，不取代既有輸出。
- 若 API 沒有回傳任何結果（找不到 / 隨機取圖失敗），維持現有的黃色提示文字，不彈出視窗。
- 新增 `-NoPreview` 開關參數：預設不加此參數時維持跳出視窗（新預設行為），加上 `-NoPreview` 時則只輸出終端機文字、不彈出視窗，供自動化 / 不需要畫面的情境使用。
